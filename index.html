How I work

The same four habits show up in every project above — they're what an analyst role actually asks for.

Define the metric that drives the decision

I find the number that actually changes an outcome — earnings per mile, follow-up coverage — not the vanity metric that just looks good.

Validate every number against source records

Nothing gets reported until it reconciles with the system of record. Two independent Lyft records agreeing within 5.6% is the standard, not the exception.

Design repeatable rules and processes

A one-off answer isn't a system. I turn a decision into documented rules and a cadence anyone can run the same way twice.

Communicate for non-technical stakeholders

The output is a clear story — a dashboard, a one-page case, a plain-English rule — that a hiring manager or teammate can act on without me in the room.
